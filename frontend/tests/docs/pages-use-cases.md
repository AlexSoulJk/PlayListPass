# Сценарии тестирования страниц авторизации

## Роутинг и guard-логика
- Неавторизованный пользователь открывает `/app` и перенаправляется на `/auth/login`.
- Авторизованный пользователь открывает `/auth/login` и перенаправляется на `/app`.
- Пользователь без `registerDraft.email` открывает `/auth/register/password` и перенаправляется на `/auth/register/email`.

## Стартовая страница
- Кнопка «Войти» переводит пользователя на `/auth/login`.
- Кнопка «Регистрация» переводит пользователя на `/auth/register/email`.

## Страница логина
- Ошибка `INVALID_CREDENTIALS` от backend отображается на уровне поля пароля.
- Ошибка не дублируется отдельным блоком вне поля.

## Страницы регистрации
- Страница email-шагa показывает сообщение `USER_ALREADY_EXISTS` в поле email.
- Страница шага пароля рендерит поля, если email есть в draft-состоянии.

## Защищенная страница
- Страница отображает email текущей сессии.
- Кнопка «Выйти» вызывает logout и возвращает пользователя на стартовую страницу.
